# 树莓派 CM4 / CM5 硬件改版设计文档（可开工版）

## 1. 目标与约束

### 1.1 目标
基于 CM4/CM5 + 自研底板实现与现有报警器板同等核心能力：
- 8 路继电器控制
- 多路开关量输入（INPUT_1/2/3、OPEN_DOOR、WATER）
- RS485 通讯
- 2 路风扇控制
- 温湿度/加速度采集
- 4G 联网（USB 4G 网卡）
- 板载以太网（CM 模块原生 MAC + 外置 PHY）
- SD 卡启动（不要求 SPI Flash 启动）

### 1.2 约束
- GPIO 扩展采用树莓派生态兼容器件
- 4G 采用 USB 网卡方案，不复用 UART/SIM 复杂控制
- 软件栈沿用 Linux 用户态服务（便于从 Pi4/Pi5 方案迁移）

---

## 2. 总体架构（CM4/CM5）

### 2.1 系统分区
- **计算核心**：CM4 或 CM5
- **底板控制总线**：I2C、SPI、UART、GPIO 中断
- **高压/大电流区**：继电器、交流采样、风扇功率驱动
- **通信区**：千兆以太网 PHY + RJ45、USB 4G、USB-RS485

### 2.2 功能分配原则
- 批量 IO（继电器/输入）交给 I2C 扩展芯片
- 风扇 PWM 交给 I2C PWM 扩展芯片
- 4G/RS485 优先 USB 化，降低引脚压力
- CM 只保留：总线、中断、少量状态控制

## 4. CM4/CM5 底板接口预算（建议）

> 注：CM4 与 CM5 在底板引脚细节存在差异，以下为“功能接口预算”，原理图阶段按官方硬件设计指南做最终 pin mux 对齐。

| 接口类型 | 建议数量 | 用途 |
|---|---:|---|
| I2C | 1 组（SDA/SCL） | MCP23017、PCA9685、AHT20、LIS3DH |
| SPI | 1 组（MOSI/MISO/SCLK/CS） | 计量/ADC（可选） |
| GPIO 输入 | 2~4 | IO 扩展中断、风扇故障、总故障 |
| GPIO 输出 | 2~4 | 系统灯、蜂鸣器、4G 电源使能 |
| USB Host | ≥2 口 | USB 4G、USB-RS485 |
| 以太网 | 1 口 | 外置 PHY + RJ45 |

## 5. I2C 地址规划（与 Pi4/Pi5 方案兼容）

| Device | AddressHex | Function | Required | Notes |
|---|---|---|---|---|
| MCP23017_1 | 0x20 | Relay outputs | Yes | 16-bit GPIO expander |
| MCP23017_2 | 0x21 | Digital inputs (INPUT/OPEN_DOOR/WATER) | Yes | INT to CM GPIO |
| PCA9685 | 0x40 | Fan PWM expansion | Yes | CH0/CH1 用于双风扇 |
| AHT20 | 0x38 | Temperature/Humidity | Yes | I2C sensor |
| LIS3DH | 0x18 | Accelerometer | Yes | 备选地址 0x19 |
| ADS1115 | 0x48 | Analog acquisition | No | 需要模拟采集时启用 |

## 6. 关键电路设计建议

### 6.1 电源树
- 12V 主输入
  - -> DCDC 5V（CM 模块与 USB 口，建议预留裕量）
  - -> 继电器/风扇电源
  - -> 3.3V 逻辑电源（外设）
- 4G 支路建议独立负载开关（受 GPIO 控制）

### 6.2 继电器/输入隔离
- 继电器线圈电源与 CM 核心电源分区
- 外部输入端建议：TVS + 限流 + 光耦/比较器
- 与 CM 连接前统一降至 3.3V 逻辑域

### 6.3 通信与抗干扰
- RJ45 侧按 PHY 参考设计完成差分与 ESD
- USB 4G、USB-RS485 端口加共模/ESD 防护
- RS485 终端匹配与偏置按现场拓扑可配置

## 7. CM4 与 CM5 版本差异策略

| 项目 | CM4 版本 | CM5 版本 | 设计策略 |
|---|---|---|---|
| 计算性能 | 中等 | 更高 | 软件同架构，镜像分配置 |
| 高速接口裕量 | 标准 | 更充足 | 底板优先统一，预留兼容焊盘 |
| 电源设计裕量 | 常规 | 建议更高 | 5V 电源按 CM5 峰值留冗余 |
| 兼容方式 | 单独底板或跳线配置 | 同一底板可兼容 | 通过 BOM/阻值档位做版本化 |

建议做一块“**共用底板**”，通过以下实现 CM4/CM5 双兼容：
- 可选阻值网络（strap）
- 可配置电源档位
- 接口复用焊盘与 0Ω 跳线

## 8. 软件与驱动落地

### 8.1 Linux 设备层
- I2C 扩展芯片：`mcp23017`、`pca9685`
- 传感器：`aht20`、`lis3dh`
- USB 4G：`ModemManager/NetworkManager` 或 `qmi`
- USB-RS485：`/dev/ttyUSB*`

### 8.2 服务拆分
- `io-service`：继电器与输入
- `fan-service`：PWM 与风扇状态
- `sensor-service`：温湿度/加速度
- `net-4g-service`：4G 管理与重连

## 10. 施工清单（可执行）

1. **原理图**：完成 CM 核心最小系统、电源树、I2C 扩展、继电器与输入隔离
2. **PCB**：按“计算区/功率区/接口区”分区布线，先完成电源与地回流
3. **EVT**：先点亮 CM 与网络，再验证 I2C 外设、继电器、输入、风扇、485、4G
4. **DVT**：做温升、浪涌、ESD、长稳压测与掉电恢复

---

## 11. 结论

采用 CM4/CM5 + 底板方案，可以在保持现有功能的同时，显著降低 GPIO 压力并提升系统可维护性。通过 I2C 扩展与 USB 外设策略，可快速落地并兼容后续 CM4 -> CM5 升级。
